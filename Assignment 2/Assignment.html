Counting Squares

Grid of doors


For this lab we will be programming a robot that has been dropped into a grid of rectangular rooms. Each wall of each room has been painted a different color: the North wall is Neon, the East wall is an Eggplant, the South wall is Sandy, and the West wall is Walnut. Walls between rooms have doors in them, but walls on the edge do not. All of the doors are closed initially.

The robot knows how to

  • Check N (or Check E, or S, or W) to see if there is a door on that wall of its current room;
  • Go N (or E or S or W) to move one room over; and
  • Do basic math and remember numbers.

If you ask it to Go through a wall that does not have a door, it isn’t smart enough to know it can’t so it will walk into the wall and damage itself.

We won’t be super formal in this lab. If we can tell what you are asking the robot to do, that’s good enough for us.

Activity 1: Rectangular

Suppose the robot is dropped into a rectangular (not necessarily square) grid of rooms and might start in any arbitrary room in the grid. Furthermore, some of the rooms in the rectangular grid are blocked. The robot has to traverse the grid to find the number of available rooms. Come up with an algorithm that the robot can use to figure out how many available rooms are in the grid. Further, you should also print the route followed.

  • An algorithm is a deterministic set of steps that does not require any outside information.

Once you have an algorithm you think is general (works for all size rectangles), submit it as Part 1.

Run Instructions and File formats

Name the top source file as robot_route.c. We will use file based I/O.
Input file: Contains the start point, grid layout and a flag variable to indicate if the room is available. The start point is given in the first line. Each room has a 4 digit location ID. First 2 digits indicate the row number and the next 2 digits indicate the column number. Flag variable is set to 1 if room available.
Output file: First line has the number of available rooms (in 4 digits). Second line indicates the route followed.
Consider, the following room layout (having 9 rooms). Please make a note of the numbering pattern of the rooms.

0000 0001 0002
0100 0101 0102
0200 0201 0202
Also, rooms 0001 and 0101 are blocked. The robot is placed initially at 0000. In such a case, the number of available rooms is 7. And the route followed is 0000, 0100, 0200, 0201, 0202, 0102, 0002. For the above example the test case input and the golden output are attached. Make sure you understand the file formats correctly and give output in the correct format.
To execute the program, we shall use the following run command
robot_route <input_file_name> <output_file_name>
Consider the run command, robot_route in.txt out.txt. This command reads the grid layout and the available rooms from in.txt and gives the output in out.txt. In order to generate the executable, robot_route from robot_route.c please write a Makefile. Using make command we will generate the executable.

Submission Instructions

  • Log on to moodle
  • ZIP the Makefile and robot_route.c
  • Name the ZIP: <First_Name>_<Entry_Number>_2D.zip and submit

Activity 2: 3D Rectangle/Cuboid

Suppose the robot is dropped into a 3D Rectangle/Cuboid grid of rooms and might start in any arbitrary room in the grid. Furthermore, some of the rooms in the rectangular grid are blocked. The robot has to traverse the grid to find the number of available rooms. Come up with an algorithm that the robot can use to figure out how many available rooms are in the cuboid. Further, you should also print the route followed.

Submit an algorithm for this case as Part 2

Run Instructions and File formats

There is a change in the room numbering conventions (as compared to 2D). To include the Z-Dimension we have additional 2 digits in the room numbering (added at the start). Consider, a 2 layer 3D structure. The bottom layer numbering is shown below:

000000 000001 000002
000100 000101 000102
000200 000201 000202

The top layer numbering is shown below:

010000 010001 010002
010100 010101 010102
010200 010201 010202

The test case input and the golden output of an 8 room (total), 2 layer structure is attached. Make sure you understand the numbering format. The first entry in the output file is the number of available rooms (6 digits).

Submission Instructions

  • Log on to moodle
  • ZIP the Makefile and robot_route.c
  • Name the ZIP: <First_Name>_<Entry_Number>_3D.zip and submit
